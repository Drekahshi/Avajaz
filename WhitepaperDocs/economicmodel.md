# AVAJAZ: Economic Model
### Incentive Architecture for a People-Powered CBO Forest Economy



---

## 1. Economic Philosophy

AVAJAZ is built on one principle: **ecological value must translate into economic value for the people who create it.**

Kenya's CFA user groups generate real ecological value — sequestered carbon, restored watersheds, preserved biodiversity, sustainably harvested forest products. Under current systems, this value is either unmeasured, uncaptured, or captured by intermediaries rather than communities. AVAJAZ builds the incentive architecture that routes this value back to its source.

The AVAJAZ economic model does not create artificial value. It reveals, documents, and connects existing value to markets willing to pay for it. The protocol's role is infrastructure, not subsidy.

---

## 2. The Three Economic Layers

### Layer 1 — Verification Economy
The base layer. CBO members earn AVAX by contributing accurate verification work. This is the primary income entry point — accessible to every registered CFA user group member with a smartphone.

**Value Flow:**
```
Ecological activity occurs (tree planted, hive maintained, seedlings propagated)
        ↓
CBO member submits verified evidence
        ↓
Peer verifier confirms
        ↓
AVAX reward distributed automatically
        ↓
Platform integrity increases → NFT asset value increases → repeat
```

**Economic Justification:** Verification work has real market value. ESG buyers, carbon registries, and certification bodies pay auditors to confirm sustainable practices. AVAJAZ distributes this audit value directly to community members who perform it at scale, replacing expensive third-party auditors with a distributed, incentivized community network.

### Layer 2 — Asset Economy
The value amplification layer. Verified activities are minted as NFTs — digital assets with documented provenance that command premium prices in sustainability markets.

**Value Flow:**
```
Verified CBO activity → NFT minted
        ↓
NFT carries auditable sustainability proof
        ↓
Commodity buyer pays 30–200% premium for verified origin
        ↓
Carbon registry issues credits against sequestration data
        ↓
Conservation funder grants against biodiversity documentation
        ↓
CBO group captures value previously lost to intermediaries
```

**Economic Justification:** The premium is not created by AVAJAZ — it already exists in sustainability markets. Specialty honey buyers, fair-trade certifiers, EUDR-compliant importers, and carbon project developers are actively seeking verified community-produced assets. AVAJAZ builds the documentation infrastructure that unlocks access to these existing premiums.

### Layer 3 — Engagement Economy
The retention and compounding layer. Four game systems convert idle verification earnings into growing assets. Players never lose principal — they only grow it through skill, accuracy, and cooperation.

**Value Flow:**
```
Verification reward earned → Staked into game system
        ↓
Skill / accuracy / coordination → Reward grows
        ↓
Reputation increases → Game tier advances → Higher rewards accessible
        ↓
Shadow Syndicate access → Governance weight → DAO participation
```

**Economic Justification:** Retention is economically critical. A platform that pays one-time rewards and loses members is financially unsustainable. The engagement economy converts transaction-based participation into relationship-based participation — members with growing stakes and advancing reputations are invested in platform success.

## 3. Incentive Mechanics Per Actor

### 3.1 CBO Group Members (Verifiers and Custodians)

| Action | Reward | Mechanism |
|---|---|---|
| Register activity (tree/hive/batch/zone) | Stake locked; returned on successful verification | StakingContract |
| Submit primary verification evidence | 70% of verification reward pool | RewardDistribution |
| Confirm as peer verifier | 20% of verification reward pool (split among peers) | RewardDistribution |
| Activity reaches Productized state | 55% (custodian) / 25% (peer verifiers) of mint reward | NFTMintContract |
| Correct chess outcome | Stake growth (skill-dependent) | GameController |
| Correct prediction market forecast | Stake growth (accuracy-dependent) | GameController |
| Chrono Siege territory hold | 2–7% passive yield per 7-day cycle | GameController |
| Shadow Syndicate treasury growth | Proportional treasury share | GovernanceContract |

### 3.2 CFA User Groups (Group-Level Treasury)

| Source | Allocation | Use |
|---|---|---|
| Verification reward split | 10% of all member verification rewards | Group equipment, training, coordination |
| NFT mint reward split | 10% of all group activity mints | Group investment fund |
| Secondary sale royalties | 60% of 5% royalty | Long-term group asset building |

**CFA Treasury Design:** Group-level treasuries are managed by group leaders through a simple multi-sig wallet. This is the first step toward full DAO governance — CBO groups manage real collective funds before the protocol transitions to on-chain governance. The treasury mechanic teaches cooperative financial management at the group level before it is required at the protocol level.

### 3.3 Protocol Treasury

| Source | Allocation |
|---|---|
| Verification reward split | 0% (Phase 1 — all goes to members and CFA) |
| NFT mint reward split | 10% |
| Secondary sale royalties | 15% of 5% royalty |
| Platform fees (Phase 2+) | Configurable |

**Protocol Treasury Use:**
- Oracle agent operational costs
- Game reward pool funding (Phase 1: primary reward source)
- Smart contract audit and security
- Ecosystem grants for CFA onboarding
- Future governance token liquidity (Phase 3)

## 4. Reward Emission Logic

### Phase 1 — Growth Stage (Years 1–2)
- Protocol treasury funds game reward pools directly
- Verification rewards funded from pilot grant funding and early platform fees
- High emission rates to incentivize early CBO adoption
- Goal: habit formation and network density, not profitability

### Phase 2 — Sustainability Transition (Years 3–5)
- Game reward pools increasingly funded by:
  - NFT mint fees (growing as verified asset volume increases)
  - ESG reporting subscription revenue
  - Enterprise API access fees
- Protocol treasury emissions decrease as revenue-backed rewards increase
- Emission schedule tied to verified activity milestones, not time alone

### Phase 3 — Revenue-Backed Stage (Years 6+)
- Game rewards fully funded by platform revenue streams
- No inflationary protocol emissions
- Reward rates stabilize around sustainable yield levels
- AVAJAZ Governance Token introduced (utility-driven: governance + staking multiplier)

**The Critical Transition:** AVAJAZ is designed from inception to move from grant/protocol-funded incentives to revenue-backed ecosystem distribution. This transition — from "printing rewards" to "distributing earned revenue" — is what distinguishes AVAJAZ from extractive play-to-earn models that collapse when early funding runs out.

## 6. Token Strategy

### Phase 1 — AVAX (Testnet → Mainnet)
- All rewards denominated in AVAX
- No custom token
- Simplicity, credibility, regulatory clarity
- Focus: prove verification mechanics and engagement retention

### Phase 2 — AVAJAZ Governance Token (Year 4–5)
**Purpose:** Governance participation, staking multiplier, treasury voting  
**Not for:** Speculation, fundraising, reward inflation

**Token Utility:**
- Governance: vote weight in DAO proposals proportional to token held + staked
- Staking multiplier: token stakers earn 1.1–1.5x base verification reward rate
- Proposal rights: minimum token threshold required to submit DAO proposals
- Treasury access: token-weighted voting on ecosystem grant allocation

**Token Distribution (indicative):**
```
CBO Member Rewards:       40% (earned through verification + games)
CFA Institutional Reserve: 20% (managed by CFA treasuries)
Ecosystem Development:    20% (grants, oracle funding, partnerships)
Team + Advisors:          15% (4-year vest, 1-year cliff)
Protocol Reserve:          5% (emergency, liquidity)
```

**Supply Control:** No mint outside of schedule. Emission tied to verified activity milestones. Burning mechanism: portion of platform fees used to buy and burn tokens (Phase 3).

### Phase 3 — AVAJAZ Impact Subnet Gas Token (Year 6+)
Dedicated subnet with custom gas token. Enables:
- Government (KFS) node participation at near-zero cost
- Enterprise ESG API integrations without C-Chain gas exposure
- Institutional-grade SLA for verification data
- Custom governance rules for African forest economy context

## 7. Supply-Demand Equilibrium

**Demand Drivers (push price up, increase token utility):**
- Growing verified asset volume (more NFTs minted → more mint fees → more reward pool)
- ESG market demand for African sustainable commodities
- Carbon credit issuance against verified tree data
- Governance token required for DAO participation
- Enterprise API subscribers paying in token

**Supply Controls (prevent inflation):**
- Rewards only issued for verified, valid ecological activity
- Reputation gating limits reward access to established actors
- Staking requirements for verification rights lock circulating supply
- Phase 3 burn mechanism removes tokens from circulation

**Stability Mechanism:** During Phase 1–2, AVAX denomination insulates the reward economy from governance token price volatility. CBO members earn stable AVAX rewards regardless of governance token market conditions.

---

## 8. Long-Term Economic Sustainability

AVAJAZ does not depend on donor funding or token speculation for long-term survival. The economic model reaches sustainability when:

1. **Verified asset premiums** cover CBO member opportunity cost — members earn more through AVAJAZ participation than alternative income sources
2. **Platform revenue streams** cover operational costs — oracle infrastructure, smart contract maintenance, team
3. **Network effects** make AVAJAZ the default verification layer for KFS-registered CFAs — switching costs exceed any competing platform benefit

The sequence: Grant-funded pilot → Fee-generating growth → Revenue-backed maturity → Community-governed infrastructure. Each stage funds the next without requiring external capital injection.
